# クエリテンプレートライブラリの選定

Find, Insert, Update のようなよく使うクエリを自動で生成したい。既存ツールから適したものを選定する。

## 要件

- よく使うクエリは自動で定義してほしい
- テーブル定義からモデルを自動生成してほしい
- テーブル（model）ごとに独自のメソッドを定義できる拡張性

## mercari/yo

[mercari/yo](https://github.com/cloudspannerecosystem/yo)

### Pros

- 指定したデータベースからモデルを自動生成してくれる
- テンプレート機能があるので、よく使うクエリの自動生成は可能

### Cons

- model の定義と同じファイルにデータベースの操作が一緒になっているので、model にカスタムメソッドを生やすことが難しい

## 結論

ColorSing では `mercari/yo` を採用してモデル生成・定型クエリ生成に使っている。カスタムメソッドを生やしたいケースは別ファイルで拡張する運用。
